# Zip Radius Finder

A production web app for advertising professionals to find all US zip codes within a given radius of a list of client zip codes — for use in Meta ad targeting campaigns.

Everything runs client-side in the browser. No backend, no database, no authentication required.

---

## Features

- **CSV upload** — drag-and-drop or click-to-browse; auto-detects the zip code column
- **Radius selector** — 1–100 mile slider with quick-select preset buttons (5, 10, 20, 30, 50 mi)
- **Haversine distance** — accurate great-circle distance calculation for all ~42,000 US zip codes
- **Async chunked processing** — batches of 50 input zips with `setTimeout` keep the UI responsive; animated progress bar shows real-time status
- **Deduplicated CSV export** — sorted ascending, single `zipcode` column, named `zip_radius_Xmi_results.csv`
- **Preview panel** — shows first 30 result zip codes inline
- **Privacy-first** — all computation happens in your browser; no data leaves your machine

## Setup

### Prerequisites

- Node.js 18+
- npm

### 1. Install dependencies

```bash
npm install
```

### 2. Build the zip code database

**This step is required before first use** (and after any source data updates).

```bash
npm run build:zipdata
```

This downloads the free public zip code dataset from GitHub, extracts zip, lat, and lng columns, and writes `public/zipcodes.json` (~1.3 MB). The file is committed to the repo so you only need to rerun this if you want to refresh the data.

### 3. Start the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### 4. Production build

```bash
npm run build
npm start
```

## Deploying to Vercel

1. Push the repo to GitHub (make sure `public/zipcodes.json` is committed — it's static data, not a build artifact).
2. Import the repo on [vercel.com](https://vercel.com).
3. No environment variables needed.
4. Deploy.

> **Note:** `public/zipcodes.json` must be present in the repo. It is not generated at Vercel build time. Run `npm run build:zipdata` locally and commit the file before deploying.

## CSV Format

The app accepts any CSV with a zip code column. The column is auto-detected by matching common header names:

| Recognized headers |
|---|
| `zip` |
| `zipcode` |
| `zip_code` |
| `postal_code` |
| `postal` |

If no header matches, the first row is scanned for a 5-digit value. ZIP+4 suffixes (`90210-1234`) are automatically stripped to `90210`. If auto-detection fails, the user can manually enter the column name.

**Example input CSV:**

```csv
client_name,zip_code
Acme Corp,90210
Widget Co,10001
Sample LLC,60601
```

**Output CSV:**

```csv
zipcode
10001
10002
10003
...
```

## Performance Notes

- The zip database (~1.3 MB JSON) is lazy-loaded only when the user clicks "Find zip codes" — not on page load.
- After the first search, the database is cached in memory so subsequent runs are instant.
- Input zips are processed in chunks of 50 using `setTimeout` between chunks, keeping the browser UI responsive even with thousands of input zip codes.
